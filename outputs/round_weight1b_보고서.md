## 2. 스키마 확장 3종 동작

### 2.1 `by_section` (도면1 동별)
주심도가 본체(사용자 결정). `(2동)기둥주심도`에서 MC1·MC2·MC3·SC1 카운트·규격 추출,
section="2동" 행에 보존. 부호도(동일 값)는 중복으로 무시.

### 2.2 `skip` (도면1 1동)
`skip: true` → SkipRow 1개 방출, 총중량 합계 미포함. CSV 첫 데이터 행에 사유 명시:
*"1동에 골구도/입면도/단면도 시트 부재로 기둥 길이 산출 불가 (정답지 명시)"*.

### 2.3 `count_override` (도면2)
익명 블록 내부 분리 TEXT 한계(baseline-5)로 카운트 측정 0 → 정답값(SC1=10, SC2=4)
격리. `count_provider` 미호출, count 컬럼에 `(override:N)`·비고 `count_override` 표기.
**정답지 검증**: `가,나동 1층 구조평면도` 시트 카운트 = SC1=10, SC2=4 → override 값과 일치.

## 3. 도면2 spec_from 검증 결과 (작업 2.2)

yaml 추정값 `"가,나,다동 1층 구조평면도"`는 **실제 매칭 시트 목록에 존재하지 않는 오기**였다.
spec_extractor 를 5개 도면2 dxf 에 직접 돌려 확인한 결과:

- `가,나동 1층 구조평면도` → SC1=H-250x125x6.0x9.0, SC2=H-200x100x5.5x8.0 (**정답 일치**)
- 지붕 구조평면도 → SG/SB (보 부재, 범위 외)

→ `dedup_routing.yaml` 의 도면2 `spec_from` 을 `"가,나동 1층 구조평면도"` 로 정정(§7.2 허용).
또한 spec_provider 를 count>0 게이트 없이 spec_from 시트에서 **직접 추출**하도록 전환 —
count_override 부호(측정 카운트 0)도 규격을 얻는다. 도면4 결과는 직접 추출도 동일값(불변).

## 4. 알려진 측정값 차이

- **도면5 길이**: 명세는 "측정 10000 vs 정답 10500" 을 우려했으나, 현재 `measure_drawing('도면5')`
  는 **10500mm 산출 = 정답 일치** — 1b 시점에 불일치 없음(명세 노트는 해소된 stale).
- **단면적 식**: KS D 3502 표값 대비 -2~3% 작음(필렛 무시). 도면4 합계 -2.9%(1a)로 확인된 알려진
  한계. 단면적 식 정확화는 별도 라운드.
