# 라운드 규격-1 보고서 — 부호별 규격 추출 + 시각화 검증

## 1. 라운드 범위 (재정의)

- **한 것**: 도면1~5에서 부호↔규격 페어링을 **전수 보존**(dedupe 제거)으로 추출하고,
  각 건에 **출처**(source_sheet, source_table_title)를 부여한 뒤 시각화·CSV·회귀
  테스트로 검증.
- **안 한 것**: 중량 산출(`total_weight`)·단위중량 룩업(`unit_weight`). 이 라운드
  실행 경로(CLI·시각화·테스트)에서 **호출하지 않는다**. 코드는 **삭제하지 않고
  보류**(LLM 라우팅 라운드 이후 재개).

근거: 중량 = 개수 × 길이 × 단위중량인데, **개수**와 **규격 키**에 중복 문제가 있어
(아래 §4) "어느 값을 칠지"는 LLM 라우팅이 결정할 일이다. 결정 전에 곱하면 이중
카운트가 난다. → **도구는 측정만, AI는 결정만.**

## 3. 도면별 추출 결과 (전수 보존)

| 도면 | 추출 건수 | 고유 (동,부호) | 일람표 구조 | 출처(source_table_title) |
|---|---|---|---|---|
| 도면1 | 44 | 24 | 1동(2종)·2동(4종) 2벌 | (없음 — 주심도/부호도, 일람표 텍스트 부재) |
| 도면2 | 6 | 6 | MEMBER LIST | `MEMBER LIST` |
| 도면3 | 32 | 22 | 4컬럼 C/P | (없음) |
| 도면4 | 6 | 4 | 1층·지붕층 일람표 분리 | `기둥 일람표`/`보 일람표` |
| 도면5 | 17 | 14 | 4컬럼 C/P | (없음) |
| **합계** | **105** | **70** | | |

- 추출 건수 > 고유 키: dedupe 제거로 같은 부호가 여러 위치에서 보존됨(의도된 결과).
- **도면4 SC1**: 1층·지붕층 **2건**으로 보존, `source_sheet`로 구분
  (`1층구조평면도` / `지붕층 구조평면도`). 길이-4에서는 dedupe로 1건만 남았었음.
- **출처 판별 규칙**:
  - `source_sheet` = 부호 좌표에서 2D 최근접 시트 제목(평면도/단면도/주심도 등).
  - `source_table_title` = 부호보다 **위에 있는** 일람표 제목 중 2D 최근접
    (일람표 헤더가 표 위에 붙고 좌우로 여러 표가 세로로 쌓이므로, "위쪽 제한"으로
    한 칸 아래 표의 헤더를 잘못 집는 문제를 해결).

## 4. 다음 LLM 라우팅 라운드의 입력 — 발견한 중복 함정

- **개수 중복**: 같은 부재가 여러 도면/시트에 표기됨.
  - 도면4: 1층 SC1 = 지붕층 SC1 = **같은 관통 기둥**. 합산하면 이중 카운트.
  - 도면1: 기둥주심도와 부호도가 같은 기둥 중복 표기.
- **규격 중복**: 같은 부호가 여러 일람표에 등장(현재 도면은 규격 동일하나, 위치별
  상이 가능).

→ 본 라운드는 이 중복을 **합치지 않고 출처와 함께 전부 보존**했다. "어느 위치
값을 진짜로 칠지"는 LLM 라우팅이 `source_sheet`/`source_table_title`을 보고
결정한다.

## 7. 중량 코드 상태

`total_weight.py` / `unit_weight.py` / `config/unit_weight_table.yaml`은 **보류**.
삭제하지 않았고, 본 라운드 실행 경로에서 호출하지 않는다. LLM 라우팅으로 개수·규격
중복 판별이 끝난 뒤 재개한다.
