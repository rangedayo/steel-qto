## 1. 이번 라운드 발견 + 수정 (3건, 전부 일반화된 보완)

도면1은 가장 복잡한 도면답게 신규 이슈 3건을 드러냈고, 모두 **도면2~5 회귀를 유지하는
일반화된 보완**으로 해결했다(본선·yaml 무수정).

### 1.1 동 라벨 분리 (`sheet_name_matcher`, `small_drawing_pipeline`)

- **증상**: 정답지 시트명은 `(1동)기둥주심도-1(...)` / `(2동)기둥주심도`처럼 동 라벨이
  붙지만, dxf 표제부에는 동 라벨이 없다(예: "기둥주심도"). partial 매칭이 첫 후보(1동)를
  잡아 **2동 파일이 1동 시트로 오매칭**.
- **수정**: 파일명에서 `(\d+동)`를 추출해 `match_sheet(dong=)`로 전달 → 시트·length 라벨
  후보를 해당 동으로 먼저 좁힌다. 동 라벨이 시트명에 안 쓰인 도면(도면2~5)은 좁힐 후보가
  없으므로 **전체 후보 유지 = 기존 동작 불변**.

### 1.2 일람표 차감 이중차감 교정 (`small_drawing_pipeline._count_columns`)

- **증상**: 도면1 부호도/주심도는 **raw 카운트가 이미 정답**(예: 2동 MC1 raw=15=정답).
  그런데 일람표 차감이 페어 수만큼 **무조건** 빼서 15→14, FAIL.
- **근본 원인**: 도면5는 일람표 범례의 부호 텍스트가 배치로 **함께 카운트**되어 과다카운트
  → 차감이 옳았다. 도면1은 범례가 별도 위치라 **애초에 카운트 안 됨** → 차감하면 이중차감.
- **실측 근거**: 페어의 `symbol_coord` ↔ 카운트 좌표 최근접 거리

  | 도면 | 거리 | 해석 |
  |---|---|---|
  | 도면3·4·5 | **0.0** (정확 일치) | 범례 부호가 배치로 카운트됨 → 차감 정당 |
  | 도면1 | **5,636 ~ 22,488** | 범례가 별도 위치, 카운트 안 됨 → 차감 부당 |

- **수정**: 차감을 **"페어 부호 좌표가 카운트된 배치 좌표와 겹칠 때(`eps=1.0`)만"**으로
  정밀화. 0.0 vs 수천 단위라 임계값 여유가 크다. 도면3·4·5는 전부 dist=0.0 → **차감 동작 불변**.

### 1.3 골구도 길이 라우팅 (`sheet_name_matcher`)

- **증상**: 2동 Y01/Y03,Y05 골구도가 `[count]`로 라우팅돼 길이 6000mm 측정이 안 됨.
- **근본 원인**: 같은 골구도 시트가 count 답안지에도 존재하며 **보 부호(VG1·SG1)**를 가진다.
  count 행이 비어있지 않아 기존 placeholder 우선 로직이 발화하지 않았다. (한 물리 시트가
  "보 카운트 소스"이자 "기둥 길이 소스" — 매처가 시트당 1 kind만 허용하는 구조적 긴장.)
- **수정**: `match_sheet(column_symbols=)` 추가 + `_count_is_placeholder()` — count 행에
  **기둥 부호가 0개**면(보뿐) 기둥 스코프에선 placeholder로 보고 length 라벨을 우선.
  partial 분기에도 동일 우선 로직 추가. `column_symbols=None`(기존 호출)이면 "행이 비었는가"만
  보므로 **도면2~5 동작 불변**.
