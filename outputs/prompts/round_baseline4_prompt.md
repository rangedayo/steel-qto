## 3. 설계 원칙

### 3.1 본선 무수정 + baseline-2/3 모듈 재사용

baseline-2/3에서 만든 모듈을 **그대로 재사용**. 신규 코드 없음.

수정이 필요한 경우:
- 매칭 충돌(1층바닥 vs 중간1층바닥)이 자동으로 풀리지 않으면 → `sheet_name_matcher`에
  일반화된 보완. 도면4·5 회귀 PASS 유지가 절대 조건.
- 다층 합산 길이가 안 잡히면 → 본선 길이 측정 함수 호출 인자 점검, 함수 수정 X.

### 3.2 매칭 충돌 처리 (가장 중요)

**예상되는 충돌 시나리오**:
- 표제부 텍스트 "1층바닥 구조평면도" → 정답지 "1층바닥 구조평면도"와 **exact match**
- 표제부 텍스트 "중간1층바닥 구조평면도" → 정답지 "중간1층바닥 구조평면도"와 **exact match**

→ 둘 다 exact match면 충돌 없음. partial fallback으로 빠지지 않는 한 안전.

**만약 충돌이 발생하는 경우**:
- exact match가 둘 다 후보로 잡히는 케이스 → 정규화 후 **정확히 동일한 문자열**만 exact로 인정
- 부분 포함은 exact가 아니라 partial로 강등 (이미 그렇게 되어 있을 가능성 큼)
- 정답지의 시트명 리스트에 "중간1층바닥 구조평면도"가 있으면 "1층바닥 구조평면도"의
  부분 포함 매칭을 차단

### 3.3 다층 길이 측정

도면3 종단면도의 길이는 19060mm. 1층 8000 + 2층 8000 + 박공 ~3000 같은 합산.

기존 길이 측정 함수가 "세로 DIMENSION 최댓값"이라면:
- 19060 자체가 단일 DIMENSION으로 그려져 있는지 → 그럼 그대로 잡힘
- 여러 DIMENSION으로 분할되어 있다면(8000 + 8000 + 박공) → 최댓값이 8000 또는 박공 길이
- 분할 케이스에서는 **합산이 필요**한데, 이건 기존 측정 함수가 안 하는 작업

**검증 방법**:
1. 도면3_종단면도계단단면도.dxf의 DIMENSION 분포 먼저 확인
2. 19060이 단일 값으로 등장하면 → 그대로 PASS
3. 분할된 값들로만 등장하면 → 정답지의 19060을 직접 측정으로 얻을 수 없음.
   라운드 한계로 보고하고 fix 안 함 (다층 합산은 다음 라운드 후보)

### 3.4 비표준 규격 추출

C1=H-600x407x20x35, C2=H-428x407x20x35는 KS D 3502 표준 외 규격. 베이스라인
컨텍스트의 `unit_weight_calc.py`가 단면적×밀도로 계산하므로 단위중량은 산출 가능.
다만 **이번 라운드는 규격 추출만 검증** — 정답지 비고의 규격과 spec_extractor 결과가
일치하는지.

### 3.5 "AI는 결정만, 도구는 측정만"

LLM·랜덤·외부 호출 0건. 결정론 보장.
