## 3. 설계 원칙

### 3.1 본선 무수정 + baseline 모듈 재사용

신규 코드 없음 원칙. 기존 모듈로 도면1 동작 검증 우선.

**동 라벨 처리가 자동으로 안 풀리면** → `sheet_name_matcher`에 일반화된 보완:
- 정답지 시트명에서 `(N동)` 패턴 추출 → 매칭 키에 포함
- 표제부 텍스트에서 `(N동)` 추출 → 매칭 키와 비교
- 동 라벨 일치 + 시트명 일치 = 정확 매칭

도면4/5/3/2 회귀 PASS 유지가 절대 조건.

### 3.2 1동 길이 N/A 처리

1동 작은 도면 8개에 길이 측정 시도하면 안 됨 (정답지 측정 불가).

처리 옵션:
- length_routing.yaml에 1동 시트가 등록 안 되어 있으면 자동 N/A
- 만약 길이 측정 함수가 강제 호출되면 결과 = None으로 처리, "측정 불가" 표시
- 정답지의 length = null과 매칭 → PASS

1동에서 측정값이 우연히 나오면 (예: 보복도에서 보 길이 잡힘) **결과 영향 없는 무관 측정**으로 처리. 카운트 측정엔 영향 X.

### 3.3 부호도-주심도 중복 카운트

1동 주심도-1: MC1=12 ✓
1동 부호도-1: MC1=12 ✓ (자동 dedup 안 함)

두 시트가 별도로 정답지에 명시되어 있으므로 시트별로 PASS 판정. baseline-2/3/4 패턴 그대로.

### 3.4 평면도에 부호 있는 케이스

(2동)SL+4.0M구조평면도: SC1=2.

기존 차감 룰이 일람표 페어 수만큼 차감하는데, 이 시트는 일람표 없이 본체만 2개. 차감 룰이
페어 0건이면 차감 안 함 → PASS. baseline-4에서 검증된 패턴.

### 3.5 Y01/Y03,Y05 골구도 길이

도면5 Y1축열골조도가 [count] 라우팅 한계를 가졌던 패턴. 도면1도 같은 한계 가능성:

- 정답지 라우팅에 (2동)Y01열골구도, Y03,Y05열골구도가 길이 소스로 등록되어야
- baseline-5 콤마-split 버그 수정이 "Y03,Y05열골구도" 시트명 처리에 도움
- length 라우팅 안 되면 [count]로 라우팅, 결과는 0=0 PASS지만 명세 의도와 다름
- 결과대로 보고만, fix 안 함 (다음 라운드 통합 처리 후보)

### 3.6 "AI는 결정만, 도구는 측정만"

LLM·랜덤·외부 호출 0건. 결정론 보장.
