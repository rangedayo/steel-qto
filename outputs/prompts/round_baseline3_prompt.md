## 1. 배경

### 1.1 베이스라인-2 결과 (도면4)

도면4 작은 도면 5개에서 측정 3종 PASS:

- 카운트 2/2 (1층 SC1=14·SC2=4 / 지붕층 0·0)
- 길이 3/3 (단면도 9000mm)
- 규격 1/1 (SC1·SC2)

이 과정에서 **신규 룰 1개**가 도입됐다:

> **일람표 차감 룰**: 작은 평면도는 일람표 부호 종수가 적어(도면4는 2종) 영역 검출
> 휴리스틱(≥4종)이 안 먹힘. 대신 `spec_extractor`가 추출한 부호↔규격 페어 수
> (=일람표 정의행)만큼 카운트에서 차감. 도면4에서 15→14·5→4로 PASS.

본선·yaml·파라미터 무수정 원칙은 지켰지만, **이 룰이 도면4에서만 검증됐다.**

### 1.2 도면5의 신규 검증 포인트

| # | 검증 포인트 | 왜 도면4와 다른가 |
|---|---|---|
| 1 | **일람표 차감 룰의 일반화** | 도면5는 부호 4종(C1~C4). 영역 검출 휴리스틱(≥4종)이 작동할 가능성 있음. 차감 룰과 영역 검출이 **둘 다** 적용되면 이중 차감 위험 |
| 2 | **C/P 합성 부호 표기** | 정답지 비고에 `C3/P3` 같은 슬래시 결합 표기 명시. spec_extractor·counter의 슬래시 매칭이 작동하는지 |
| 3 | **DIMENSION 0개 도면** | `도면5_정면도우측면도.dxf` (= 입면도1)는 DIMENSION 엔티티가 0개. 길이 측정 함수가 이 케이스를 안전하게 처리하는지 |
| 4 | **Y1 축열 골조도** | 정답지 라우팅에 길이 측정 소스로 명시. 일반적인 단면도와 다른 골조도 형식 |

### 1.3 정답지 핵심 값 (도면5)

**카운트 (`도면5-기둥` 시트)**:
- `1층바닥 구조평면도`: C1=2, C2=4, C3=8, C4=6 (합 20)
- 나머지 평면도 6개: 모두 0 (이중카운트 방지 검증 대상)

**길이 (`도면5-기둥-길이` 시트)**:
- C1~C4 모든 인스턴스 = **10500mm**
- 측정 소스: 주단면도1, 주단면도4 (직접 측정), Y1 축열 골조도 (cross-check)

**규격 (`도면5-기둥-길이` 시트 비고)**:
- C1 = H-300x300x10x15
- C2 = H-250x250x9x14
- C3 = H-450x200x9x14 (`C3/P3` 표기로 등장 가능)
- C4 = H-200x200x8x12

## 3. 설계 원칙

### 3.1 본선 무수정 + baseline-2 재사용 우선

baseline-2 모듈(`sheet_title_extractor`, `sheet_name_matcher`,
`small_drawing_pipeline`, `visualize_small`)은 **그대로 재사용**한다.

신규 모듈을 만들지 말고, **인자·옵션 조정**으로 도면5 케이스를 처리한다.
만약 baseline-2 모듈에 케이스별 분기가 필요해서 수정해야 한다면:

- 도면5 검증 결과가 깨졌을 때만 수정
- 도면4 회귀(`test_baseline2_regression.py` 19 PASS)가 깨지지 않도록 보장
- 차감 룰이 도면5에서 깨지면 **차감 룰 자체를 재설계**하는 것이 옳음 (도면4 결과는
  유지하면서) — 단일 케이스 대응 보다는 일반화된 룰이 목표

### 3.2 차감 룰 일반화 시나리오

도면5에서 가능한 시나리오 3가지:

| 시나리오 | 결과 | 대응 |
|---|---|---|
| **A. 차감 룰만으로 PASS** | 일람표 자동 검출은 안 작동, 차감 룰이 4종에서도 정확히 4 차감 → PASS | 그대로 정착. 도면4와 같은 흐름 |
| **B. 영역 검출 + 차감 룰 = 이중 차감** | 카운트가 정답보다 적게 나옴 (예: C1=1, C2=3...) | 차감 룰을 "영역 검출 결과가 0건일 때만 발동" 조건부로 보완 |
| **C. 둘 다 안 먹힘** | 카운트가 정답보다 많게 나옴 (예: C1=3, C2=5...) | 차감 룰 트리거 조건 재검토. spec_extractor 페어가 일람표만 잡는지 확인 |

**판단 근거 데이터**: 도면5_1층바닥구조평면도 실행 시 (a) 원시 카운트, (b) 영역
검출 결과, (c) spec_extractor 페어 수, (d) 차감 후 결과를 로깅. 시나리오 식별
후 대응.

### 3.3 DIMENSION 0개 도면 안전 처리

`도면5_정면도우측면도.dxf`는 DIMENSION 0개 (사전 검증됨). 길이 측정 함수가
이 케이스에서 **에러 없이 `length=None`** 반환하도록.

정답지에서 입면도1은 길이 측정 소스가 아니므로(라우팅에 주단면도1·주단면도4·
Y1축열 골조도만 명시) `length=None`이 정답. 시각화 HTML도 깨지지 않게.

### 3.4 C/P 합성 부호 처리

이미 `llm_wiki_룰북.md` §2.3에 정의됨 ("슬래시 결합 본체 부호로 인정").
기존 counter·spec_extractor가 그대로 동작하는지 검증만. 동작 안 하면 보고만
하고 fix는 별도 라운드.

### 3.5 "AI는 결정만, 도구는 측정만"

LLM·랜덤·외부 호출 0건. 결정론 보장.

매칭 실패 케이스 있으면 보고하고 fallback yaml에 케이스만 등록 (보편 룰 우선).

### 작업 2 — 도면5 측정 3종 실행

baseline-2의 `small_drawing_pipeline`을 도면5 9개에 호출:

1. 카운트: 1층바닥 구조평면도만 비-0, 나머지 8개 0
2. 길이: 주단면도1·4·Y1축열골조도 측정, 나머지 측정 안 함 또는 None
3. 규격: 1층바닥 구조평면도에서 C1~C4 추출

**각 시트별 원시 측정값과 룰 적용 결과를 로깅** — 차감 룰 동작 추적용 (3.2 데이터).

### 작업 3 — 차감 룰 일반화 판정

도면5_1층바닥구조평면도의 결과로 시나리오 A/B/C 식별 (3.2 표):

- **시나리오 A** (PASS): 그대로 진행, 보고서에 "차감 룰이 부호 4종에서도 일관 작동" 명시
- **시나리오 B** (이중 차감): 차감 룰에 조건부 트리거 보완 (영역 검출 0건일 때만).
  도면4 회귀 PASS 유지 검증.
- **시나리오 C** (둘 다 미적용): spec_extractor 페어가 본체 부호를 잘못 일람표로
  해석한 건지 분석. 트리거 조건 재설계.

수정 발생 시 도면4 회귀 19 PASS, 도면5 검증 PASS 둘 다 유지.

### 작업 4 — C/P 합성 부호 검증

도면5_1층바닥구조평면도에서 C3가 `C3/P3` 표기로 등장하는지 dxf 인스펙션. 등장한다면:

- 카운트에서 `C3/P3` 본체가 C3로 인정되는가
- spec_extractor에서 `C3/P3`의 규격이 정확히 매칭되는가
- 카운트 8개에 `C3/P3` 인스턴스가 포함되는가

PASS면 보고. FAIL면 별도 이슈로 기록(이번 라운드 fix 안 함).

### 작업 5 — DIMENSION 0개 도면 처리

도면5_정면도우측면도.dxf 호출 시:

- 길이 측정 함수가 에러 없이 None 반환
- HTML 시각화가 깨지지 않고 정상 생성 (DIMENSION 없이도)
- 정답 비교에서 "측정 대상 아님"으로 표시 (FAIL 아님)

### 작업 6 — Y1 축열 골조도 길이 검증

도면5_Y1축열골조도.dxf의 길이 측정값이 10500mm와 일치하는지:
