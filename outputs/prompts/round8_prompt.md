# 작업: 라운드 8 — 도면3 기둥 추가 (슬래시 패턴 매칭 확장 + height 필터 yaml 추가)

## 이번 라운드 목표

도면3의 기둥 부호(`C1`, `C2`, `C3`, `C4`)를 회귀 테스트에 추가한다. 도면3 기둥은 본체에서 `C{n}/P{n}` 슬래시 결합 형식으로 표시되므로, `match_symbol` 부분일치 매칭에 슬래시 단어 경계를 추가하는 작업이 핵심이다.

**도면1·2·4의 기존 회귀 결과(6 PASS / 2 FAIL)를 정확히 유지하는 것이 절대 조건.**

도면5, 보 부호, height 필터 자동화는 이번 라운드 범위 밖.

## 배경 — 도면3 사전 진단으로 확인된 사실

도면3 DXF를 사전 진단한 결과:

```
modelspace 직접 TEXT/MTEXT:
  'C1':    1회   (height 276, layer=SYM)  ← 일람표
  'C2':    1회   (height 276, layer=SYM)  ← 일람표
  'C3':    1회   (height 276, layer=SYM)  ← 일람표
  'C4':    1회   (height 276, layer=SYM)  ← 일람표
  'C1/P1': 8회   ← 본체 (정답 C1=8과 일치)
  'C2/P2': 15회  ← 본체 (정답 C2=15와 일치)
  'C3/P3': 8회   ← 본체 (정답 C3=8과 일치)
  'C4/P4': 1회   ← 본체 (정답 C4=1과 일치)
  'C1-600X407X20X35': 1회  ← 규격 안내
```

도면3 정답지(C1=8, C2=15, C3=8, C4=1)는 **일람표 빼고 본체만 카운트한 값**. `C{n}/P{n}` 카운트가 정답과 정확히 일치하므로, 슬래시 매칭만 풀리면 회귀 통과 가능.

**핵심 발견**: 도면3의 정답값은 **C-T-L 결합 표기(`C1/P1`)에서 C 부분만 추출**한 값이다. 이는 도면2의 `BR2 L-80X80X7` → `BR2` 매칭(라운드 2)과 동일한 부분일치 패턴이지만, **경계 문자가 공백·하이픈이 아닌 슬래시**라는 점이 다르다.

## 작업 1 — 사전 진단 스크립트 (코드 변경 전)

**먼저 진단부터.** 룰 변경 전에 다음을 측정·기록한다.

`poc_v2/tests/investigate_도면3.py` 신규 작성:

### 1-A. height 분포 측정

도면3 modelspace TEXT/MTEXT 의 height 분포 출력. 부호 패턴 매칭되는 텍스트만 대상.

기대: height 222 (본체) / 276 (작은 글자) 두 무리로 갈리는지 확인. `min_height` 추천값 산출 (예: 250).

### 1-B. 슬래시 패턴 통계 (도면별 비교)

도면1·2·3·4 전체에서 `C{n}/P{n}` 또는 `^[A-Z]+\d+/` 패턴이 얼마나 등장하는지 카운트.

기대: 도면3 에만 집중적으로 등장, 도면1·2·4 에는 거의 없거나 0. **만약 도면1·2·4 에도 다수 등장한다면 슬래시 매칭 룰 확장이 기존 회귀를 깨뜨릴 위험이 있음** — 그 경우 작업 2를 중단하고 멘토 결정 요청.

### 1-C. 신호 2·3 자동 판정 결과

도면3 에 대해 `auto_detect_policy()` 가 어떤 결정을 내리는지 출력:
- `exclude_table_regions`: ON / OFF
- `exclude_with_spec`: ON / OFF
- 판정 근거 (일람표 후보 영역 수, 규격 패턴 텍스트 비율)

기대: 둘 다 ON. 그래야 일람표 4개와 `C1-600X407X20X35` 규격 텍스트가 자동으로 빠지고 본체 32개만 남음.

만약 OFF로 판정되면 정답지와 카운트 비교가 어긋남 — 신호 2·3 임계값 조정 필요할 수 있음. 그 경우 진단 결과만 기록하고 코드 변경 보류.

### 1-D. 슬래시 매칭 룰 확장 시뮬레이션

`match_symbol` 룰을 임시로 슬래시까지 확장한다고 가정하고, 도면3 baseline 카운트가 정답과 일치하는지 시뮬레이션. 실제 `counter.py` 는 건드리지 않고, 진단 스크립트 안에서만 가짜로 적용.

기대 결과:

```
도면3 기둥 시뮬레이션 (슬래시 매칭 + 신호 2·3 자동 ON):
  C1: 예측 8 / 정답 8 → PASS
  C2: 예측 15 / 정답 15 → PASS
  C3: 예측 8 / 정답 8 → PASS
  C4: 예측 1 / 정답 1 → PASS
```

**모든 결과를 outputs/round8_사전진단.md 에 저장.** 멘토 보고용.

## 작업 2 — 매칭 룰 확장 (사전 진단 통과 시에만)

작업 1 결과가 다음 조건을 모두 만족할 때만 작업 2 진행:
- 도면1·2·4 에 슬래시 패턴이 없거나 매우 적음
- 도면3 에 신호 2·3 이 자동 ON 으로 판정됨
- 시뮬레이션에서 도면3 기둥 4종 전부 PASS

### 2-A. `counter.match_symbol` 수정

현재 부분일치 매칭의 단어 경계 문자에 슬래시(`/`)를 추가한다. 라운드 2 에서 도입한 패턴 ("BR2 L-80X80X7" → "BR2") 의 자연스러운 확장.

**중요 — MC10 오매칭 방지 룰 유지**: 부호 뒤 첫 글자가 숫자면 매칭 실패. 슬래시 추가 후에도 이 룰이 깨지지 않는지 확인 (예: `C1/P1` 에서 `C1` 매칭 시 `/` 가 숫자가 아니므로 통과).

### 2-B. `symbol_rules.yaml` 에 도면3 min_height 추가

작업 1-A 에서 산출한 추천값으로 한 줄 추가. 예:

```yaml
text_height_filter:
  도면1:
    min_height: 177
  도면2:
    min_height: 302
  도면3:
    min_height: 250   # 추가
  도면4:
    min_height: null
```

**실제 값은 작업 1-A 진단 결과에 따라 결정.** 진단에 나온 height 분포를 보고 본체(큰 글자)와 작은 글자 사이 컷을 정함.

### 2-C. 회귀 테스트 확장

`test_regression.py` 의 호출 한 줄 수정:

```python
# 변경 전
_TOTALS = drawing_symbol_totals(
    category="기둥",
    drawings=["도면1", "도면2", "도면4"],
)

# 변경 후
_TOTALS = drawing_symbol_totals(
    category="기둥",
    drawings=["도면1", "도면2", "도면3", "도면4"],
)
```

## 절대 조건 — 회귀 안전망

작업 2 적용 후 `pytest -v` 결과:

- **도면1 기둥 4종 (MC1, MC2, MC3, SC1) — 전부 PASS 유지 (절대 조건)**
- **도면2 기둥 SC1·SC2 — 라운드 6 baseline 그대로 FAIL 유지** (분리 TEXT 미해결 항목, 라운드 6 D항)
- **도면4 기둥 SC1·SC2 — 전부 PASS 유지 (절대 조건)**
- **도면3 기둥 4종 (C1, C2, C3, C4) — 전부 PASS** (이번 라운드 신규 통과)

기대 결과: **10 / 12 통과** (라운드 7 6/8 → 라운드 8 10/12). 도면2 SC1·SC2 두 건만 알려진 FAIL.

이 중 하나라도 어긋나면 룰 확장이 의도치 않은 부작용을 일으킨 것. 즉시 롤백하고 원인 분석.

1. **변경 전 baseline 저장**: 현재 `pytest -v` 결과 텍스트 저장 (6 PASS / 2 FAIL)

4. 조건 미충족이면 작업 2 중단하고 보고. 충족 시에만 작업 2 진행.
