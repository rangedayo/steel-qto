## 3. 줄바꿈 결합 시트명 매칭 (작업 1)

- 정답지 카운트 시트는 행 단위로 로드되며, `가,나동 1층 구조평면도` 가 단일 행으로 존재
  → 표제부 `가,나동 1층 구조평면도` 와 **exact 매칭**. 줄바꿈(`\n`) 결합 변형은
  `가,나동 입면도\n(가,나동 정면도, 가,나동 좌측면도)` 처럼 입면도 행에 나타나며,
  입면도 통합/분리본 표제부 `가,나동 입면도` 가 **partial 매칭**(부분 포함)으로 풀림.
- **종단면도 unmatched**: `도면2_가나동종단면도.dxf` 는 표제부 텍스트가 추출되지 않음
  (title=None). 카운트 0·길이 비대상이라 결과 영향 없으나 매칭은 실패 — 한계로 기록.

## 4. 본선 FAIL vs 작은 도면 결과 (★ 이번 라운드 메인 발견)

| 부호 | 본선 큰도면 (predict) | 작은도면 (이번) | 정답 | 개선? |
|---|---|---|---|---|
| SC1 | **0** | **0** | 10 | **NO** |
| SC2 | **0** | **0** | 4 | **NO** |

**작은 도면 입력 모델은 본선 1단계 FAIL을 자연 해결하지 못했다.** 둘 다 0/0.

### 근본 원인 (작업 4 — 블록 INSERT 점검)

`도면2_가나동1층구조평면도.dxf` 의 기둥은 다음 구조:

- **modelspace**: `'SC1'`·`'SC2'` 통짜 TEXT 가 각 1개씩 존재하나 height **300.987 < 302**
  (도면2 min_text_height 필터)로 제거됨. 게다가 이 둘은 MEMBER LIST **정의행**이라
  설사 잡혀도 차감 룰로 빠질 값.
- **블록 INSERT**: 실제 기둥 배치 ~14개는 익명 블록(`*B465`~`*B472` 등) INSERT 인데,
  **각 블록 내부에 `'SC'`(h400)와 `'1'`(h280)이 분리된 별도 TEXT** 로 들어있다.
  `match_symbol('SC')` → None, `match_symbol('1')` → None → **재결합 불가** → 0.
- ATTRIB 없음.

→ **입력 단위(큰 도면 vs 작은 도면)와 무관한 counter.py 카운트 메커니즘 한계.**
counter 는 INSERT 의 ATTRIB·블록정의 TEXT 를 보지만, 한 블록 안에 부호가 글자
단위로 쪼개져 있으면(`SC`+`1`) 공간 인접성으로 재결합하는 로직이 없다.
**counter.py 수정은 7.1 절대 금지** → 이번 라운드는 한계로 문서화.

### PoC 가치 측면 해석

- 작은 도면 입력 전환이 **모든** 본선 한계를 자동 해결하는 것은 아님이 확인됐다.
- 단, FAIL 원인이 "입력 모델"이 아니라 "특정 도면의 분리 TEXT 블록 표기"로 **정확히
  국소화**됐다는 점은 성과 — 일반화된 split-TEXT 재결합기(블록 내 인접 글자 병합)를
  baseline 레이어에 추가하면 본선 무수정으로도 해결 가능(다음 라운드 후보).

## 5. 차감 룰 (작업 3-2)

- 1층 spec_extractor 페어 = {SC1:1, SC2:1} (MEMBER LIST 정의행 인식 OK).
- 그러나 raw 카운트가 이미 0(블록 분리 TEXT 미집계)이라 차감 결과도 0.
  raw [0,0] → 페어 차감 → [0,0]. 차감 룰 자체는 정상이나 **입력 카운트가 비어 무의미.**

## 7. 매칭 보완 1건 — length 라벨 콤마-split

**파일**: `poc_v2/baseline2/sheet_name_matcher.py` (`load_length_labels`)

**문제**: `length_routing` 의 도면2 sheet_label `"가,나동 횡단면도"` 를 기존 `re.split(r"[,、]")`
가 콤마로 쪼개 `['가', '나동 횡단면도']` 로 만듦 → 표제부 `가,나동 횡단면도` 가 어느
토큰과도 exact 안 맞아 length 라우팅 실패(빈 placeholder 카운트 행으로 빠짐).

**보완**: 리스트 구분자를 **"콤마+공백"(`, `) 또는 `、`** 로 한정. 콤마가 토큰 내부
(`가,나동` = 가동·나동 결합)에 쓰일 수 있으므로 공백 없는 콤마로는 쪼개지 않음.

```python
for token in re.split(r",\s+|\s*、\s*", str(raw)):
```

**일반화 근거 / 무영향**: 도면4 `"종단면도, 횡단면도"`·도면5 `"주단면도1, 주단면도4"`·
도면3 `"종단면도, 계단단면도"` 는 모두 콤마+공백이라 2토큰 그대로 유지(회귀 19/33/19
PASS 확인). 도면1 `"...Y03,Y05열골구도"` 의 토큰 내부 콤마도 이제 보존.
